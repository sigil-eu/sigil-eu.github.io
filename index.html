The Problem

MCP Has No Security Model

AI agents execute real-world actions ‚Äî reading emails, sending money, accessing databases. The dominant protocol (MCP) provides none of this:

MCP: Missing

No Identity Verification

Any process can call any tool. No way to know who is behind an agent action.

MCP: Missing

No Content Scanning

API keys, IBANs, PII flow freely between agents and tools with zero interception.

MCP: Missing

No Audit Trail

No record of what tools were called, by whom, or what data was accessed.

MCP: Missing

No Permission Gating

No risk levels, rate limits, or confirmation flows for destructive actions.

How SIGIL Works

Five Traits. Any Backend.

SIGIL defines interfaces ‚Äî you bring the implementation. Regex scanner, HSM vault, LDAP identity? All plug in.

IdentityProvider

ü™™ Identity Binding

Bind users to verifiable trust levels ‚Äî OIDC, eIDAS, SSI. Gate tools by who you are.

SensitivityScanner

üîç Content Interception

Detect API keys, IBANs, PII before they reach the LLM. Domain-specific patterns.

VaultProvider

üîí Encrypted Vault

Intercepted secrets are encrypted and stored with opaque pointers. Never leak.

AuditLogger

üìã Tamper-Evident Audit

Every security event logged with HMAC signatures. Who did what, when, why.

SecurityPolicy

üõ°Ô∏è Policy Enforcement

Risk classification, rate limiting, confirmation gates. Zero-trust by default.

SigilMcpServer

üîå MCP Server

Quick Start

Secure an MCP Server in 4 Lines

Add sigil-protocol to your Cargo.toml, implement a scanner, and wrap your tools.

use sigil_protocol::mcp_server::{SigilMcpServer, ToolDef}; use sigil_protocol::TrustLevel; // Create a SIGIL-secured MCP server let scanner = Arc::new(MyScanner); let audit = Arc::new(MyAuditLogger::new()); let mut server = SigilMcpServer::new("my-tools", "1.0", scanner, audit); // Register tools ‚Äî SIGIL scans input AND output automatically server.register_tool(my_tool); // Trust-gate sensitive tools (requires eIDAS identity) server.register_tool_with_trust(banking_tool, TrustLevel::High); // Every call is now identity-gated, scanned, and audited let response = server.handle_request(request, caller_trust).await;
Conformance Levels

Choose Your Security Depth

SIGIL defines three conformance levels. Start minimal, add layers as needed.

SIGIL-Core

Identity + Audit

Know who did what, when. The minimum baseline.

"We need basic accountability for our agent actions."

SIGIL-Guard

Core + Scanner + Vault

Full content interception. Sensitive data never leaks to the LLM.

"We handle PII and financial data ‚Äî nothing must leak."

SIGIL-MCP

Guard + MCP Server

Every MCP tool call is identity-gated, scanned, and audited.

"We run MCP servers in production ‚Äî every call must be secured."
